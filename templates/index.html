如何启用Clash API

检测到Clash正在运行，但无法连接到API。请根据您使用的Clash客户端启用API：

Clash for Windows
打开Clash for Windows
点击左侧菜单中的"设置"（齿轮图标）
在"开发者设置"区域，确保"允许局域网连接"已启用
记下"控制台端口"的值（默认为9090）
如果显示了"API密钥"，也请记下这个值
在本页面的"设置"部分，更新API地址为 http://127.0.0.1:端口号 并保存
注意：程序将自动尝试从配置文件 %USERPROFILE%\.config\clash\config.yaml 中读取API配置
Clash.Meta
打开Clash.Meta的配置文件（config.yaml）
确保配置中包含以下内容：
external-controller: 127.0.0.1:9090
secret: "你的密钥"（可选）
保存配置文件并重启Clash.Meta
在本页面的"设置"部分，更新API地址和密钥
Clash Verge
打开Clash Verge
点击左下角"设置"图标
选择"系统设置"选项卡
找到"混合控制端口"（通常是9090）
在本页面的"设置"部分，更新API地址为 http://127.0.0.1:端口号
OpenClash
登录到OpenWrt管理界面
进入"服务"->"OpenClash"->"全局设置"->"控制设置"
确保"启用控制"已勾选
查看"控制端口"（默认为9090）
在本页面的"设置"部分，更新API地址为 http://路由器IP:端口号
手动修改配置文件
Clash的配置文件通常位于：
  • %USERPROFILE%\.config\clash\config.yaml
  • %APPDATA%\Clash for Windows\config.yaml
在配置文件中找到或添加以下配置：
external-controller: 127.0.0.1:端口号
secret: "你的密钥"（可选）
保存配置文件并重启Clash
在本页面的"设置"部分，填写相同的API地址和密钥

提示： 本程序会自动尝试从常见位置检测Clash API配置，但如果您使用了非标准配置或修改过默认端口，可能需要手动配置。
